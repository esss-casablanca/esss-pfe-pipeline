# Closure Status Rubric

Apply the **minimum honest** status. Closure is proven by a citable V2 location, never by
assertion.

## Status definitions

- **closed** — the required action was fully performed in V2 and is visible at a cited V2
  location. No fabricated value was used. For a `needs_new_search` issue, the real search was run
  and its result is reflected in V2.
- **partially_closed** — the action was performed as far as available information allowed, but
  some element still depends on student/supervisor input or a value that does not yet exist. The
  remaining element is marked and named.
- **cannot_close_yet** — closing the issue would require fabricating data, a source, an ethics
  detail, or a count the student has not supplied. Insert the appropriate marker, set this status,
  and state exactly what the student must provide. **Never** convert this into a fake *closed*.

A Critical or Major issue left in neither *closed* nor a legitimately blocked state means V2 is
not ready.

## What counts as closure, by concern code

| Code | Closure evidence required in V2 |
|---|---|
| `QUES` | Precise, framework-anchored question now stated |
| `FRAM` | Correct framework adopted and justified; criteria realigned |
| `PRISMA` | Correct standard applied; flow reported; any invented count removed or replaced by a real count / `[COUNT REQUIRED]` |
| `STRUCT` | Missing section/table added (cite the table letter) |
| `SRCH` | Flagged Consensus queries run and logged (Table A) |
| `TRIA` | Added/justified sources; Table B completed |
| `ELIG` | Explicit pre-specified criteria with reasons; PFE feasibility criteria present |
| `CIT` | Core citations verified; unverified excluded/labelled; Table I updated |
| `EXTR` | Study-level re-extraction across required fields; abstract-limited marked |
| `METHCOMP` | Methodological-comparison table (Table E) present before synthesis |
| `HETERO` | Heterogeneity assessed; synthesis method justified (Table F) |
| `ROB` | Design-appropriate tool applied with judgement and consequence (Table G) |
| `SYNTH` | Synthesis integrates and answers the nine synthesis questions; no incompatible merging |
| `CERT` | GRADE/CERQual or structured confidence assessment added |
| `INTERP` | Claims narrowed to the certainty/directness of the evidence |
| `GAP` | Each gap now specifies what, where, for whom, why |
| `ORIG` | Contribution over existing reviews articulated |
| `PFE` | SUSPENDED under R1 — standalone article; no Component-2 implications required |
| `FEAS` | SUSPENDED under R1 — no empirical-feasibility requirement |
| `ETH` | Implied design is non-invasive and ethically realistic |
| `CTX` | Moroccan/Francophone/institutional relevance added where it matters |
| `JTGT` | Emphasis realigned to a plausible target journal family |

## Reminder

If a `cannot_close_yet` or `partially_closed` issue is Critical or Major, make it unmistakable to
the student in the closing summary — these are the items that block a clean second appraisal.
