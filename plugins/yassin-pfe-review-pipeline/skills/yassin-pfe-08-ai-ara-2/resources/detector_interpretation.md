# External Detector Interpretation (Modes B/C)

Use only when the student supplies a report from Copyleaks, Compilatio Magister+,
iThenticate 2.0 / Turnitin AI Writing Detection, or Originality.ai. **Never imply any of these was
run unless the student supplied its report.**

## Intake — extract from each report
Tool name · report date · document title/version · language the tool detected · overall score or
category · highlighted passages · tool-specific categories · thresholds (if visible) · whether the
report indicates AI-generated, AI-paraphrased, similarity, semantic similarity, or translation
similarity · whether screenshots/PDF pages are included.

## Interpretation principles
- A detector score is **probabilistic**; it is never proof of misconduct and never a guarantee of
  clearance.
- Interpret each score **in context**: document type, section, language, and health-science
  conventions. Formulaic sections (methods, ethics, inclusion criteria, statistics) can inflate
  apparent AI-likeness legitimately.
- Do **not** reclassify the text purely from the external report; relate it to the internal audit.
- Separate evidence types clearly: external (user-supplied), internal inference, combined
  concordance, or inconclusive/discordant.
- Translation-like patterns are common in Francophone authors writing in English — flag for
  verification, not as proof.

## Hard limits
Never convert a score into a misconduct conclusion. Never describe how to lower a score. Never
state the text is "cleared", "undetectable", or "safe from detection". The aim is legitimate
authorship strengthening and informed human interpretation only.
